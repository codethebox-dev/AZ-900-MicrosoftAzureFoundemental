# Glossary
{: .mt-0 }

Key terms and definitions for the AZ-900 exam.

---

| Term | Definition |
|---|---|
| **Availability Set** | A logical grouping of VMs to protect against hardware failures within a datacenter. Provides 99.95% SLA. |
| **Availability Zone** | A physically separate datacenter within an Azure region with independent power, cooling, and networking. Provides 99.99% VM SLA. |
| **Azure Arc** | A set of technologies for managing resources across on-premises, multi-cloud, and edge environments from Azure. |
| **Azure Firewall** | A managed, cloud-based network security service that protects Azure Virtual Network resources. |
| **Azure Functions** | A serverless compute service for running event-driven code without provisioning or managing infrastructure. |
| **Azure Policy** | A service for creating, assigning, and managing policies that enforce rules on Azure resources. |
| **Azure Resource Manager (ARM)** | The deployment and management layer for Azure resources. Allows you to create, update, and delete resources using templates. |
| **Blob Storage** | Azure's object storage solution for unstructured data (images, videos, documents, backups). |
| **CapEx** | Capital Expenditure — upfront spending on physical infrastructure. On-premises model. |
| **Compliance Manager** | A tool in Microsoft Purview that helps organisations assess, monitor, and improve compliance posture. |
| **Conditional Access** | Entra ID feature that enforces if-then access policies based on signals like user location or device state. |
| **Defender for Cloud** | Microsoft's unified security posture management and threat protection platform. |
| **Elasticity** | The ability to automatically scale resources up or down to match workload demand. |
| **Entra ID** | Microsoft's cloud-based identity and access management service (formerly Azure Active Directory). |
| **GRS** | Geo-Redundant Storage — 6 copies of data across two Azure regions. |
| **High Availability (HA)** | The ability of a service to remain accessible even when components fail. Expressed as % uptime SLA. |
| **Hybrid Cloud** | A deployment model that combines public and private cloud environments. |
| **IaaS** | Infrastructure as a Service — cloud provider manages hardware; customer manages OS and above. |
| **Key Vault** | Azure service for securely storing secrets, encryption keys, and certificates. |
| **LRS** | Locally Redundant Storage — 3 copies of data within a single datacenter. |
| **Management Group** | A container for managing access, policy, and compliance across multiple subscriptions. |
| **MFA** | Multi-Factor Authentication — requires two or more verification factors to sign in. |
| **Network Security Group (NSG)** | A filter for inbound and outbound network traffic to Azure resources, based on rules. |
| **OpEx** | Operational Expenditure — ongoing spending for services. Cloud model. |
| **PaaS** | Platform as a Service — provider manages infrastructure + OS; customer manages application and data. |
| **Private Cloud** | Cloud infrastructure operated exclusively for a single organisation. |
| **Public Cloud** | Cloud infrastructure owned and operated by a third-party provider and shared among customers. |
| **RBAC** | Role-Based Access Control — grants access to Azure resources based on assigned roles. |
| **Region** | A geographical area containing one or more Azure datacenters connected by a low-latency network. |
| **Region Pair** | Two Azure regions within the same geography paired for disaster recovery. |
| **Resource Group** | A logical container for Azure resources. All resources belong to exactly one resource group. |
| **SaaS** | Software as a Service — provider manages everything; customer uses the application. |
| **Scalability** | The ability to increase resources to handle growing demand. Can be vertical (bigger) or horizontal (more). |
| **Secure Score** | A quantified measure of an organisation's security posture in Defender for Cloud (0–100). |
| **Shared Responsibility Model** | Framework defining which security tasks are handled by Microsoft vs the customer. |
| **Subscription** | A logical unit in Azure that provides access to Azure services and defines a billing boundary. |
| **VM Scale Sets** | A group of identical, load-balanced VMs that can scale in or out automatically. |
| **ZRS** | Zone-Redundant Storage — 3 copies of data across 3 availability zones in one region. |
